## 2. Notes

- **Graph Explorer is a read surface, not an authority.** If the backend denies a node or edge, the
  UI must not infer it from layout gaps, counts, or error messages.
- **Review actions are human attestation.** Promote/reject/relink should be explicit, auditable,
  and idempotent; accidental double-submit must not create duplicate evidence.
- **Use generated contract types only.** The graph and review APIs are already generated by M2/M3;
  web should fail to build if those contracts drift.
- **Screen size is a real risk.** Split graph layout, detail drawer, table, and actions into child
  components/composables to keep `.vue` files under the project cap.
